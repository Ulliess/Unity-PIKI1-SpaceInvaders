# friendslop co-op Space Invaders

Кооперативный мультиплеерный friendslop по мотивам Space Invaders.

## Описание

Два игрока управляют космическими кораблями в нижней трети экрана. Инопланетяне движутся сверху вниз. Корабли стреляют автоматически вверх. Задача — выжить и набрать очки.

**Визуальный стиль:** Минималистичный 2D, вдохновленный The Tower (Google Play)

## Core Gameplay

- **2 корабля** в нижней трети экрана, управление в двух плоскостях
- **Автоматическая стрельба** строго вверх
- **Столкновения** кораблей как твёрдых тел (можно "подставить" второго)
- **5 типов инопланетян** — разные размеры, формы, скорости
- **3 формы кораблей:**
  - Прямоугольник — быстрое, слабое оружие  
  - Круг — медленное, сильное оружие  
  - Треугольник — среднее по силе/скорости, AoE эффект

## Мета-геймплей

| Результат | Очки |
|-----------|------|
| Смерть | 0 |
| Победа (оба живы) | 1 |
| Победа (второй умер) | 2 |

Таблица лидеров ведётся по сумме очков.

## Фичи

- Главное меню + выбор корабля
- Создание/подключение к лобби
- Генерация волн врагов (рандомная, но сбалансированная)
- Core gameplay (движение, стрельба, столкновения, HP)
- Сетевая синхронизация (позиции, взаимодействия, события)
- Таблица лидеров
- WebGL билд (браузер — Win, Android, iOS)
- Тестирование при плохой сети (Интерполяция)

## Технологии

- **Unity** (2D)
- **Netcode for GameObjects** + Unity Relay/Lobby
- **WebGL** для браузерного деплоя

## Подключение к проекту

- [Git](https://git-scm.com/downloads) (или можно GitHub Desktop)

### Шаги

1. **Клонируем репозиторий:**
   ```bash
   git clone https://github.com/Ulliess/Unity-PIKI1-SpaceInvaders.git
   ```

2. **Открываем проект в Unity HUB:**
   - Unity Hub → вкладка **Projects**
   - **Add** (рядом с New Project)
   - Выбираем папку `Unity-PIKI1-SpaceInvaders/project`


### Рабочий процесс (Git Flow)

Каждая задача делается в **отдельной ветке**. Название ветки: `feature/что-делаем`.
`feature/` — это не папка, а просто конвенция имени.

#### Начинаем работу над задачей

```bash
# 1. Переключаемся на main и обновляем
git checkout main
git pull

# 2. Создаем ветку под задачу
git checkout -b feature/player-movement

# 3. Работаем, коммитим
git add .
git commit -m "add player movement script"

# 4. Пушим ветку на GitHub
git push -u origin feature/player-movement
```

#### Завершаем задачу

1. Заходим на GitHub → во вкладке Pull Requests жмём **New Pull Request**
2. Выбираем `feature/player-movement` → `main`
3. Пишем что сделали, жмём **Create Pull Request**
4. Кто-то из команды смотрит и жмёт **Merge**

#### Если нужно обновить свою ветку свежими изменениями из main

```bash
git checkout main
git pull
git checkout feature/моя-ветка
git merge main
```

### Правила работы с Git
- **Перед работой** — `git pull` (или `git fetch` + `git pull`)
- **Коммитим часто**, с понятными сообщениями
- Папки `Library/`, `Temp/`, `Logs/` — **НЕ коммитятся** (уже в .gitignore)
- `.meta` файлы **ОБЯЗАТЕЛЬНО коммитить** — без них Unity потеряет связи

### Правила работы с Unity

- **НЕ редактируем одну сцену (.unity) одновременно** — мёрдж конфликты в сценах практически неразрешимы
- Всё делаем через **префабы** — корабли, враги, UI элементы.
- Если нужно потестить — создаём **свою тестовую сцену** (например `Scenes/Test_ullies.unity`)
- В чужие сцены не лезем, но если очень надо — предупреждаем об этом.
